Heronix SubConnect

The first purpose-built substitute teacher app. QR code activation, network-restricted access, digital attendance, emergency plans, and automatic data wipe. Substitutes get everything they need including school emergency procedures. Schools keep everything secure. No one else has this.

Coming 2026
See How It Works View Suite Ecosystem

The Problem

Substitute teachers are the most underserved role in school operations

What Subs Get Today

  • A paper roster and printed map
  • Verbal instructions from the front office
  • No access to school digital systems
  • Can't take attendance electronically
  • No way to report issues
  • No access to student accommodations
  • Paper attendance = data gap in SIS

What SubConnect Provides

  • Digital roster with photos and seating
  • Building maps and bell schedules
  • One-tap digital attendance
  • Real-time sync to SIS
  • Complete emergency plans (fire, lockdown, evacuation routes, safe zones)
  • Emergency contacts and procedures
  • IEP/504 flags (view-only, no PII)
  • Teacher instructions for the day

How It Works

Five simple steps from arrival to departure

1

Install Once

Substitute downloads SubConnect app on their personal phone. One-time install works at any Heronix-equipped school.

2

Connect to School WiFi

App detects authorized school network and activates. Only functions on school WiFi - cannot access data outside building.

3

Scan QR Code

Scan QR code at front office or classroom. Creates temporary session for that teacher, those periods, that date only.

4

Take Attendance & Access Info

View class rosters, take one-tap attendance, see building maps, emergency procedures, bell schedules, and teacher instructions. All data syncs to SIS in real-time.

5

Auto Data Wipe on Exit

When sub leaves school network, session deactivates automatically. All cached data is wiped from device. Attendance is already synced. Ready for next assignment.

Security Architecture

Enterprise-grade security for temporary access

Network Restriction

  • App validates authorized school network before displaying data
  • WiFi SSID matching + subnet verification
  • Optional MAC address or certificate validation
  • Session suspends instantly when leaving network
  • Reconnecting resumes session (same school day)

Data Protection

  • No student data permanently stored on device
  • All data cached in encrypted memory only
  • Session data wiped when leaving network
  • Student PII tokenized through Guardian
  • IEP/504: "accommodation required - see front office" only

Access Control

  • Subs ONLY access classes they're assigned to
  • Assignment configured by admin before arrival
  • QR code valid for assigned date only
  • QR code school-specific (can't reuse at another school)
  • All activity logged in SIS audit trail

Guardian Integration

  • All data passes through Guardian Security Perimeter
  • Student tokens are session-specific
  • If phone compromised: only tokenized data exposed
  • Tokens expire when session ends
  • Zero real PII ever on substitute's device
